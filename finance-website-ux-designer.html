Case study

Redesigning a financial services website

Conducting a full overhaul of the website, bringing a new focus on clear navigation and accessibility. Designing iterative wireframes, creating prototypes for user tests, and building a design system.

1. Identifying the problems

UX maturity

Despite only being 6 years old, the website was - to put it bluntly - a mess!

The UX Design function had only been introduced the year before. This meant, when senior stakeholders asked for their niche new project to be added to the main navigation bar, there was nobody to gatekeep and to represent the user’s interests.

Consistency

Sitewide consistency was poor, leaving the user to learn new patterns and UI elements on every page.

For example, clicking a tile on the ‘multigrid’ would sometimes take users to another page, sometimes open up a PDF, and sometimes do nothing at all.

We saw this time and time again on HotJar clickmaps and UserTesting videos. (Never has the browser’s Back button been so popular!)

Navigation and accessibility

With no logical Information Architecture to organise content, the main navigation bar held 11 items at times, including short-lived events and links to microsites.

Meanwhile, everywhere you looked, there were accessibility issues (nightmares?): from the standard body text size of 14px to the regular use of white text over images.

Things needed to change!

2. Working alongside an agency

Before our team was hired, the company had attached an agency to the project. This meant that, when we arrived, the project was already partway through the Discovery process. Unfortunately, this had been far too focused on the opinions of senior stakeholders in the business, rather than the problems that users were encountering.

Refocusing on the user

As a team, we therefore had a week between meetings to rapidly create some alternative proposals based around our users.

Each week, I would create a handful of wireframes and we discussed their relative merits. Then our amazing UX Researcher set up unmoderated user tests to get feedback on the various proposals, including those from the agency.

As we gained the agency’s trust, we were eventually able to embed this user testing officially in the process.

3.Iterating and testing

The company's key products, funds, were presented on incredibly dense pages. Some key details were presented on the left sidebar, but due to sidebar blindess users were failing to find this content. Users had to scroll down a long(!) page with poor text hierarchy to find key content like factsheets, ISIN numbers and portfolios. This was shown in user testing - and in the Hotjar click maps:

Our UX Researcher conducted some user testing to discover user's priorities with the content. This suggested that past performance, holdings and prices were especially important to 80% of users.

We also set up a comparison between the vertical tabbed navigation proposed by the agency...

... and the possibility of horizontal tabbed navigation.

The user testing revealed that the horizontal tabbed navigation had a higher ease of completion rating of 4.6 out 5 (compared to 4.17 out of 5), and so we decided to move in this direction going forward.

4. Visual design

Giving feedback

The agency then did a great first iteration of the visual design. As my company didn't have an official in-house UI Designer, we had to limit our feedback to usability issues, rather than collaborating as a cohesive UX/UI team.

For example, some 12px font sizes crept in, and we knew our brand team were uncomfortable with adding colour without an image to set the colour's context.
