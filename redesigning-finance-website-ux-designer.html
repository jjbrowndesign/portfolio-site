Improving accessibility and navigation

Conducting a full redesign of a financial services website, led by user testing. Identifying problems, designing wireframes, creating prototypes, and iterating based on the results.

Context

Time period

July 2022 – December 2022

Team

  • UX Lead
  • UX Researcher
  • UX UI Designer (me!)
  • UX Designer – agency
  • UI Designer – agency

Problem

The company had seen a huge growth in the number of investors using its website during Covid. As the website came under increased scrutiny from new users who hadn't learned its unique patterns, the issues with consistency, navigation and accessibility were proving hard to ignore.

Opportunity

The Brand team were also working on a rebrand, so this became a perfect opportunity to apply a minimalist, clean UI to the site, maximising the benefits of the aesthetic-usability effect.

1. Identifying the problems

When I first joined the company, I conducted a heuristic evaluation of the site based on my knowledge of user-centred design principles. I then analysed HotJar clickmaps and my Research colleague watched user testing videos. We came to the same conclusions...

Consistency

Sitewide consistency was poor, leaving the user to learn new patterns on every page.

This goes against Nielsen Norman Group's principle of consistency and standards.

For example, pages were frequently dominated by grids of equal-sized tiles. When clicked, these tiles would sometimes navigate users to a subpage, sometimes take them to a different section of the site, sometimes open up a PDF, and sometimes do nothing at all!

But there were no signifiers to suggest to the user which did which...

→ We therefore decided to move away from these misleading UI elements, creating bespoke components for links (following recognisable patterns from other websites) and for documents (with signifiers to differentiate them for the user).

Navigation

Navigation was cluttered, illogical and inconsistent.

The main navigation bar held 11 items at times: stuffed full of niche projects, short-lived events and links to external microsites.

Previously there had been nobody to 'gatekeep' areas like navigation menus to make sure user experience was championed alongside competing business goals.

→ It was clear we needed to set up a tree test or card-sorting exercise to establish a logic Information Architecture. And, once established, we would need to be more protective of this menu in future.

Accessibility and inclusivity

Everywhere you looked, there were accessibility issues.

For example:

  • The body text size was 14px, rather than the recommended minimum of 16px.
  • There was white or black text over images without an appropriate overlay or outline.
  • HTML heading tags were not used or were given a wrong hierarchy.
  • Images didn't have alt text.
  • Aria-labels were nonexistant.

→ As well as adding all of these improvements into our designs, we also made sure to collaborate with teams elsewhere in the business; for example, I worked with colleagues in Brand to improve colours in charts, ensuring good contrast and differentiation.

2. Solving the problems

Refocusing on the user

The company had only just set up its UX team. So, when we arrived, the project was already partway through an unstructured discovery process that had unfortunately failed to engage with users and their problems.

As a team, we took the decision to run weekly test-and-learn sprints, so we could refocus the process back on the user without slowing the project down.

Prototyping and testing

At the start of each week, I would use tools like Mobbin and conduct some competitor analysis to create a handful of wireframes that might solve the problems relevant to that page.

These were created and prototyped in Figma (which I'm luckily pretty fast at using).

For example, I proposed three different styles of filtering for our investment strategies page:

The team would discuss their relative merits and select two especially promising options.

Next, our amazing UX Researcher would set up unmoderated usability tests on UserTesting. We would then assess the findings and data to conclude which design to progress, and I would make iterative tweaks based on any important user feedback.

Presenting validated designs to stakeholders

I would then present it back to the wider project group, and the Researcher would provide the accompanying research results to back up our designs.

This collaborative approach took a lot of hard work and trust, as we had to work quickly and relied on each other to keep the process moving.

But it meant we were able to embed user research back into a process that had almost forgotten about its users! It allowed us to demonstrate to the business the importance of validating designs, and we managed to keep up the pace of the project too.

Example test and iteration

Navigation and discoverability problems

I had concerns that our core products (investment funds) were presented on incredibly dense pages.

To test this hypothesis, I indentified some key content (factsheets, ISIN numbers and portfolios) through Hotjar click maps and stakeholder input.

I then analysed the scroll depth to reveal that users were giving up on scrolling down the long! page.

We then conducted some usability testing, which confirmed that users were failing to find this content. In some cases, this was no doubt due to sidebar blindness, as this information had been put in grey containers on the side of the page.

The poor text hierarchy also seemed to be a factor, based on the user interviews.

Navigation was clearly something we needed to solve urgently. So I created prototypes for a vertical tabbed navigation...

... and a horizontal alternative. I felt this might improve the discoverability of these key sections and improve consistency between desktop and mobile, but I was keen to hear what users thought.

Research outcomes

The user testing revealed that the horizontal tabbed navigation had a higher ease of completion rating of 4.6 out 5 (compared to 4.17 out of 5). So, after a few further tweaks to the design, we were able to present our findings to the business and validate our suggestion.

3. Applying a minimalist UI

Collaborating

The agency's UI Designer then did a great first iteration of the visual design, bringing the brand to life:

It was at this stage that I set down some key accessibility and usability requirements, based on our early heuristic evalution. For example, there was no typographic hierarchy, the button states were inconsistent, and some inaccessible font sizes had crept in.

It was then my job to take this visual design template, fix some inconsistencies, and ultimately create a Design System...
