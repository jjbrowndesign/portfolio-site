Case study

Redesigning a financial services website

Conducting a full overhaul of the website, bringing a new focus on clear navigation and accessibility. Designing wireframes, creating prototypes for user tests, and building a design system.

1. Identifying the problems

UX maturity

Despite only being 6 years old, the website was - to put it bluntly - a mess!

The UX Design function had only been introduced the year before. This meant, when senior stakeholders asked for their niche new project to be added to the main navigation bar, there was nobody to gatekeep and to represent the user’s interests.

Consistency

Sitewide consistency was poor, leaving the user to learn new patterns and UI elements on every page.

For example, clicking a tile on the ‘multigrid’ would sometimes take users to another page, sometimes open up a PDF, and sometimes do nothing at all.

We saw this time and time again on HotJar clickmaps and UserTesting videos. (Never has the browser’s Back button been so popular!)

Navigation and accessibility

With no logical Information Architecture to organise content, the main navigation bar held 11 items at times, including short-lived events and links to microsites.

Meanwhile, everywhere you looked, there were accessibility issues: from the standard body text size of 14px to the regular use of white text over images.

2. Working alongside an agency

Before our team was hired, the company had attached an agency to the project. This meant that, when we arrived, the project was already partway through the Discovery process. Unfortunately, this had been more focused on senior stakeholders than on user's problems.

Refocusing on the user

As a team, we therefore had a week between meetings with the agency to rapidly prototype some alternative proposals based around analytics and user testing.

Each week, I would create a handful of wireframes and the UX team would discuss their relative merits. Then our amazing UX Researcher set up unmoderated user tests to get feedback on the various proposals, including those from the agency. They would then share their findings and we would make iterative tweaks to the most successful wireframes.

As we gained the agency’s trust, we were eventually able to embed this user testing officially in the process.

3.Iterating and testing

The company's key products - investment funds - were presented on incredibly dense pages. Some key details were presented on the left sidebar, but due to sidebar blindess users were failing to find this content. Users had to scroll down a long(!) page with poor text hierarchy to find content like factsheets, ISIN numbers and portfolios. I indentified this content was key through the Hotjar click maps:

Our UX Researcher conducted user testing to validate this and to discover user's other priorities. The testing also suggested that past performance, holdings and prices were especially important to 80% of users.

I then set up prototypes so we could test the vertical tabbed navigation proposed by the agency...

... and the possibility of horizontal tabbed navigation, which I felt would improve the visibility of these key sections and would make for more consistency between desktop and mobile.

The user testing revealed that the horizontal tabbed navigation had a higher ease of completion rating of 4.6 out 5 (compared to 4.17 out of 5). So, after a few tweaks to the design, we were able to present our findings to the business and convinced them to move in this direction going forward.

4. Visual design

Giving feedback

The UI Designer then did a great first iteration of the visual design.

We worked together to improve some areas, especially accessibility (some 12px font sizes had crept in!).

We followed a similar process throughout the planning stage of the project, until we had mockups for all the key areas of the site. It was then time to move onto the build stage...
